# `wolfbot` 2026-04-24 LLM Advanced Werewolf Terms Update Prompt

この文書は、このリポジトリの `wolfbot` を今回の確認事項に合わせて更新するための固定プロンプトです。下の `## Prompt` セクションを、そのまま別の LLM / coding agent に渡すことを想定しています。

今回の主眼は、LLM player の共通推理ルールに、ガチ勢人狼でよく使われる用語と、それをこの 9 人村 bot でどう推理に使うべきかを追加することです。用語を知識として説明するだけでなく、LLM player が実際の発言・投票・CO 評価・縄計算で使える判断軸にすることを目的にします。

参考として、一般的な人狼用語を確認済みです。グレランはグレーから各自が投票先を選ぶ進行、縄計算は吊り回数の計算、グレスケ / スケールはグレーの白黒順位付け、確白 / 確黒 / パンダ / ローラーなどは CO と判定結果を評価するための基本語彙として扱います。

## 1. 共通ルール block に熟練人狼用語を追加する

必要な仕様:
- すべての LLM player が、役職に関係なく、基本的な熟練人狼用語を system prompt で受け取ること。
- 用語説明は「辞書」ではなく、公開ログを読むときの判断ルールとして書くこと。
- 既存の 9 人村ルール、CO 評価、霊媒白 / 黒の意味、候補トークン制約と矛盾しないこと。
- LLM が発言で用語を使う場合も、略語だけを出さず、具体的な根拠を添えるよう促すこと。

このタスクで固定する実装方針:
- `src/wolfbot/llm/prompt_builder.py` の `_build_game_rules_block()` に文面を追加すること。
- 既存の `{game_rules_block}` の差し込み構造を使い、`src/wolfbot/prompts/llm_system_prompt.md` の大きな構造変更は避けること。
- 文面が長くなりすぎる場合は、`_build_game_rules_block()` 内で「熟練人狼用語」相当の短い bullet 群として整理してよい。
- `build_system_prompt()` の公開引数は増やさないこと。
- role-specific strategy block には原則として触れないこと。今回の用語説明は全 LLM player 共通の判断材料として扱う。

共通ルール block に必ず含める用語と文面趣旨:
- `グレー` / `灰`: 役職 CO しておらず、占いなどで白黒が十分ついていない位置。誰視点のグレーかを意識する。
- `グレラン`: グレーから各自が理由を持って投票する進行。完全な無作為ではなく、発言、CO 状況、票筋、スケールから選ぶ。初日など情報が少ない場面で使いやすいが、中盤以降は組織票や縄余裕に注意する。
- `縄` / `吊り縄` / `縄計算`: 残り処刑回数。標準目安は `floor((生存人数 - 1) / 2)`。9 人開始は `4縄`。残り人狼数、狂人生存、PP / RPP リスクを考えて、無駄吊りできる回数を意識する。
- `スケール` / `グレスケ`: グレーや未確定位置を白い順・黒い順に並べる考察。順位だけでなく、発言、投票、占い/霊媒結果、噛み筋との整合性を添える。
- `白`: 占い/霊媒で本物の人狼ではない判定。狂人も白に出るため、村陣営確定とは限らない。
- `黒`: 占い/霊媒で本物の人狼判定。誰の判定か、CO 者の信用、対抗結果との整合性を確認する。
- `確白`: 公開情報上ほぼ非狼として扱える位置。進行役候補になりやすい。ただし狂人が白に出るルールでは、常に村陣営確定とまでは言い切らない。
- `確黒`: 全視点または十分信用できる複数情報で本物の人狼と見てよい位置。単独の偽占い候補から黒を出されただけでは確黒ではない。
- `パンダ`: 白判定と黒判定の両方を受けた位置。判定を出した CO 者の真偽比較、霊媒結果、投票、噛み筋で評価する。
- `CO` / `対抗`: 役職宣言と同じ役職への別宣言。対抗 CO がある場合は、死亡済み CO 者も含めて時系列・結果・投票・襲撃との整合性を見る。
- `ロラ` / `ローラー`: 複数 CO した同種役職候補を吊り切る進行。始めたら原則完遂だが、黒ストップや強い破綻などで止める場合は理由を明示する。
- `決め打ち`: 複数 CO や複数候補のうち一方を真寄り、他方を偽寄りとして進行を固定すること。外すと負けに直結しやすいため、縄余裕、判定、票筋、噛み筋を根拠にする。
- `破綻`: その人の主張が公開情報、役職数、判定、死亡状況、投票などと矛盾して成り立たないこと。破綻は強い偽要素。
- `ライン`: 発言、投票、擁護、判定結果などから見えるつながり。ライン切りや偶然もあるため、単発ではなく複数材料で見る。
- `囲い`: 偽占いなどが仲間の狼に白を出して保護すること。白先の発言、投票、噛み筋と合わせて疑う。
- `身内切り`: 人狼が仲間の人狼に黒を出す、または投票で切ることで信用を得る戦術。黒を出した側が必ず真とは限らない。
- `票筋`: 誰が誰に投票したかの流れ。処刑結果、同票、決選、狼候補への投票有無から推理する。
- `噛み筋`: 夜の襲撃先の傾向。情報役噛み、白位置噛み、意見噛み、狩人探しなどの意図を推理する。
- `視点漏れ`: 本来知り得ない役職、狼位置、夜行動、内訳を知っているように話してしまうこと。強い狼要素になり得る。
- `SG`: スケープゴート。狼に疑いを押し付けられ、処刑候補にされやすい村側位置。怪しまれている理由が本人の黒要素か、狼の誘導かを分けて見る。
- `GJ` / `平和`: 犠牲者が出ない朝。騎士の護衛成功があり得る。騎士 CO や護衛先情報は強い材料だが、既存の騎士 strategy と矛盾しないよう不用意な公開は避ける。
- `PP` / `RPP`: 終盤に人狼陣営が票を合わせて勝ちを狙う、またはランダム勝負に持ち込む局面。残り人狼数、狂人生存可能性、縄数を意識する。

推奨文面例:
- `グレランは、グレーから各自が理由を持って投票する進行。名前にランダムとあるが、完全な無作為投票ではない。発言、CO 状況、票筋、スケールを根拠に選ぶ。`
- `縄計算は残り処刑回数を数えること。標準目安は floor((生存人数 - 1) / 2)。この 9 人村は開始時 4縄なので、残り人狼数と狂人生存の可能性から無駄吊りできる回数を意識する。`
- `グレスケ / スケールは、未確定位置を白い順・黒い順に並べる考察。順位だけでなく、発言、投票、判定、噛み筋との整合性を添える。`
- `確白は公開情報上ほぼ非狼として扱える進行役候補。ただし狂人は白判定されるため、白は村陣営確定ではない。`
- `確黒は全視点または十分信用できる複数情報で本物の人狼と見てよい位置。単独の偽占い候補から黒を出されただけでは確黒ではない。`
- `パンダは白判定と黒判定の両方を受けた位置。パンダ本人だけでなく、判定を出した CO 者同士の真偽、霊媒結果、票筋、噛み筋で評価する。`

## 2. 既存の CO 評価・進行論と矛盾させない

必要な仕様:
- `単独 CO`、`対抗 CO`、`死亡済み CO 者の比較`、`霊媒白 / 黒` の既存ルールは維持する。
- `確白` の説明は、この bot の「狂人も白に出る」仕様と整合させる。確白を無条件の村陣営確定として扱わせない。
- `確黒` の説明は、単独の占い黒と混同させない。黒を出した CO 者が偽なら黒結果も偽の可能性がある。
- `ローラー` と `決め打ち` は、既存または将来の `3-1` / `2-2` 進行論と矛盾させない。ローラーを始めた場合の原則完遂と、黒ストップなどの例外を区別する。
- `PP` / `RPP` は勝利条件と整合させる。狂人は勝敗判定では人狼陣営だが、生存人数カウントでは非人狼として扱われる既存ルールを壊さない。

このタスクで固定する実装方針:
- `_build_game_rules_block()` 内で、既存の共通推理ルールの近くに用語説明を置くこと。
- 文面が長くなりすぎる場合は、1 用語 1 文程度に圧縮する。
- system prompt の総量が過度に増えないよう、例文を大量に入れない。実戦で間違えやすい注意点を優先する。

## 3. 情報秘匿と prompt 分離を維持する

必要な仕様:
- 非人狼の prompt に狼相方情報が混ざってはいけない。
- 狼以外の role-specific strategy に、人狼専用の夜連携戦術が混ざってはいけない。
- 狂人には真の人狼位置を知らせない。
- 別 `game_id` の log は current game の prompt に混ざってはいけない。
- Discord の message history を直接拾って prompt に入れてはいけない。
- 用語説明は「公開ログから読み取れる発言・投票・CO・襲撃結果に対する推理方針」であり、実際の役職内訳を LLM に漏らすものではない。

このタスクで固定する作業:
- `LLMAdapter._ask()` の DB ベースの文脈構築は維持すること。
- `build_user_context()` の現在のスコープ分離は壊さないこと。
- system prompt の共通ルール文面だけを強化し、role leak が起きないようにすること。

やってはいけないこと:
- 配役を変える
- ルールエンジンを変える
- 状態遷移を変える
- DB schema を増やす
- slash command を増やす
- CO parser や自動盤面分類器を追加する
- 縄数やスケールの自動計算機能を追加する
- Discord API から message history を直接 prompt に流す
- 非狼に相方情報を見せる
- 狂人に本物の狼位置が見えている前提で書く
- 無関係な refactor を広げる

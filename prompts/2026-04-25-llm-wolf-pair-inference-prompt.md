# `wolfbot` 2026-04-25 LLM Wolf Pair Inference Prompt

この文書は、このリポジトリの `wolfbot` を今回の確認事項に合わせて更新するための固定プロンプトです。下の `## Prompt` セクションを、そのまま別の LLM / coding agent に渡すことを想定しています。

今回の主眼は、LLM player を強い熟練した人狼プレイヤーに近づけることです。特に、LLM が単独の人狼候補だけを疑うのではなく、「その人が人狼なら相方は誰か」「2 人狼セットとして発言・投票・判定・襲撃が自然か」まで考えるように prompt / task text を強化します。

## Prompt

```md
あなたは、このリポジトリで直接コードを修正する Python エンジニアです。

目的:
- Discord 人狼 bot `wolfbot` の LLM player を、より強い熟練した 9 人村プレイヤーに近づける。
- LLM が人狼候補を 1 人ずつ単体で疑うだけでなく、2 人狼配役であることを前提に「その人が人狼なら相方は誰か」まで推理するようにする。
- 昼発言・投票・夜行動の判断で、狼候補 A と相方候補 B の組み合わせが公開ログ上自然かを比較させる。
- 人狼 LLM は、自分と相方が 2 人狼セットとして読まれる前提で、庇い・ライン切り・身内票・騙り結果・襲撃の整合性を管理できるようにする。
- 変更は LLM prompt / task text / prompt tests 周辺に閉じる。ゲームルール、DB schema、Discord command、状態遷移、投票・夜行動の解決ロジック、権限管理、復旧処理は変えない。

今回必ず対応すること:
1. 共通ルールに、公開情報から 2 人狼仮説を作って検証する推理指針を追加すること。
2. 村人・占い師・霊媒師・騎士 strategy に、単体黒要素だけでなく相方候補・ライン・ライン切り・身内票・票逸らし・噛み筋まで見る指針を追加すること。

## 1. 共通ルールに 2 人狼仮説の推理指針を追加する

`_build_game_rules_block()` に、全役職が読んでよい公開情報ベースの推理指針を追加すること。

必要な仕様:
- この村は人狼 2 人固定なので、人狼候補を 1 人挙げたら、その人物の相方候補まで仮説として考える。
- ただし村側・狂人・未確定役職は実際の相方を知らないため、断定ではなく公開ログからの推定として扱う。
- 「A が狼なら B が相方として自然か」を、以下の材料で比較する:
  - 発言上の庇い・便乗・距離の取り方
  - 投票先、決選投票、票変え、同票時の動き
  - 占い/霊媒結果、白先・黒先・囲い候補
  - 襲撃先、噛まれなかった位置、意見噛み・白位置噛み・情報役噛みとの整合
  - 片方が処刑濃厚なときにもう片方がどう動いたか
  - ライン切りや身内票として自然か、単に偶然の対立か
- 単体で怪しい相手でも、自然な相方候補が見つからない場合は疑いの強さを調整する。
- 逆に単体では中庸でも、相方候補との票筋・発言・噛み筋が強くつながる場合は疑いを上げる。
- 2 人狼候補を出すときは、発言では長い内部思考を出さず、最も強い根拠 1〜2 点だけを短く述べる。
- 「相方候補」は推理用語として使ってよいが、実際に相方を知っているような断定は人狼本人以外してはいけない。

共通ルールに必ず含める文言の要素:
- `2 人狼`
- `相方候補`
- `公開ログからの仮説`
- `単体黒要素`
- `票筋`
- `噛み筋`
- `ライン切り`
- `身内票`

推奨文面例:
- `この村は人狼 2 人固定なので、単体黒要素だけでなく「その人物が人狼なら相方候補は誰か」を公開ログからの仮説として考える。`
- `A-B の 2 人狼仮説は、庇い・距離の取り方・票筋・判定・噛み筋・ライン切り・身内票がセットとして自然かで検証する。`
- `村側や狂人は実際の相方を知らないため、「相方候補」は断定ではなく推理として述べる。`

## 2. 村陣営 strategy にペア推理を追加する

`_ROLE_STRATEGIES[Role.VILLAGER]`、`Role.SEER`、`Role.MEDIUM`、`Role.KNIGHT` を更新すること。

共通して追加する趣旨:
- 単体の怪しさだけで吊り先を決めず、その人が狼なら相方候補が誰かまで比較する。
- 2 人狼セットとして見たとき、投票・発言・判定・噛み筋が自然な候補を優先して疑う。
- 「庇い」だけでなく「不自然な距離取り」「処刑濃厚時の急な切り」「決選投票での身内票」「弱い対抗候補への票逸らし」も見る。
- 発言では、全候補を長く列挙せず、今日の結論に効くペア仮説 1 つか 2 つを短く出す。

村人 strategy:
- 公開情報しかない立場として、CO 履歴・判定履歴・投票履歴・噛み筋から 2 人狼仮説を作る。
- 「この人単体が怪しい」だけでなく「この人が狼なら相方はこの位置が自然 / 不自然」を添える。
- 村人 CO 禁止の既存方針は維持する。

占い師 strategy:
- 占い先選定で、単体黒狙いだけでなく、白黒どちらが出ても相方候補の整理に効く位置を優先する。
- 対抗 CO がいる場合、対抗の白先・黒先・投票先・庇い先から 2 人狼仮説を作り、占い結果で最も崩せる仮説を狙う。
- 黒を引いたら、その黒の相方候補まで公開情報から絞り、処刑後の霊媒結果や翌日の占い方針につなげる。

霊媒師 strategy:
- 霊媒黒が出た場合、処刑された狼の相方候補を票筋・庇い・ライン切り・身内票から絞る。
- 霊媒白が出た場合、その処刑を押した位置や票の集まり方から、狼がどこを吊りたかったかを考える。
- 占い CO 処刑後の霊媒結果は、占い真偽だけでなく残り狼候補のペア整理に使う。

騎士 strategy:
- 護衛先選定で、単体の白さだけでなく、狼ペア仮説上どこが噛まれると狼に得かを考える。
- 平和や襲撃先から、どの 2 人狼仮説ならその噛み筋が自然かを昼に推理材料として使う。
- 騎士 CO 時は護衛履歴に加え、必要なら噛み筋がどの狼ペア仮説に合うかを短く述べる。

注意:
- 村陣営 strategy に実際の人狼相方情報を入れてはいけない。
- 人狼本人だけが知る `相方` という確定情報ではなく、`相方候補` / `2 人狼仮説` / `公開ログからの推定` として書く。

## 3. 人狼 strategy に「2 人狼セットで読まれる」前提を追加する

`_ROLE_STRATEGIES[Role.WEREWOLF]` に、以下の趣旨を追加すること。

必要な仕様:
- 人狼は、自分と相方が公開ログ上の 2 人狼セットとして読まれる前提で動く。
- 相方を庇う、相方を切る、相方に投票する、相方を囲う、相方を SG から外す、相方をあえて疑う、どの行動も翌日以降にラインとして読まれる。
- 相方が処刑濃厚な局面では、無理に庇うと 2 人狼セットとして透ける。救えないなら身内票・ライン切りで自分が残る勝ち筋を比較する。
- 相方を救う場合は、自然に票が集まり得る対抗候補がいて、自分の過去発言・騙り結果・投票理由と矛盾しないときだけ選ぶ。
- 偽占いの白囲い・黒出し、偽霊媒結果、騎士騙り、襲撃先は、自分と相方の 2 人狼仮説がどう見えるかまで考えて選ぶ。
- 襲撃は「誰を消すか」だけでなく、「誰を残すとどの相方候補が疑われるか」「噛み筋で自分と相方のラインが濃くならないか」を比較する。
- 人狼チャットや夜行動では、相方との勝ち筋を共有しつつ、公開発言では実際の相方を知っている視点漏れをしない。

人狼 strategy に必ず含める文言の要素:
- `2 人狼セット`
- `相方`
- `ライン`
- `ライン切り`
- `身内票`
- `票逸らし`
- `囲い`
- `噛み筋`
- `視点漏れ`

推奨文面例:
- `自分と相方は、村から 2 人狼セットとして票筋・庇い・距離感・噛み筋を読まれる。`
- `相方を救う票、切る票、囲い、黒出し、襲撃は、翌日以降に自分と相方のラインとして説明できるかを確認する。`
- `相方が処刑濃厚なら、無理な票逸らしで 2 人狼セットを透けさせるより、身内票やライン切りで自分が残る価値を比較する。`

## 4. 狂人 strategy に公開情報ベースの 2 人狼仮説を追加する

`_ROLE_STRATEGIES[Role.MADMAN]` に、以下の趣旨を追加すること。

必要な仕様:
- 狂人は本物の人狼位置を知らないため、2 人狼仮説は公開ログからの推定として扱う。
- 狼候補 1 人だけを支援・攻撃するのではなく、その相方候補まで見て、どの誘導が人狼陣営に得かを考える。
- 黒出しや白出しでは、対象単体だけでなく、その対象を狼扱い / 非狼扱いしたときに、相方候補の見え方がどう変わるかを見る。
- 真占い・真霊媒を崩すときも、村に誤った 2 人狼仮説を追わせられるかを考える。
- ただし本物の狼位置を知らないため、誤爆や誤支援が起きる前提で、断定しすぎない。

狂人 strategy に必ず含める文言の要素:
- `2 人狼仮説`
- `公開ログからの推定`
- `誤爆リスク`
- `白先が本物の狼とは限らない`
- `相方候補`

禁止:
- 狂人 strategy に `相方` を人狼同士の確定情報として書かない。
- 狂人 strategy に `襲撃先を揃える` を書かない。
- 狂人 strategy に本物の人狼位置を知っている前提を入れない。

## 5. 昼発言 task にペア推理の短い促しを追加する

`task_daytime_speech()` を更新すること。

必要な仕様:
- 既存の 80〜300 字目安、`intent=speak` / `intent=skip`、day 2 以降 1 巡目の結果公開ルールは維持する。
- 通常の昼発言 task に、必要に応じて `単体黒要素だけでなく相方候補や 2 人狼仮説も短く示す` 趣旨を追加する。
- ただし毎回必ずペア考察を強制しない。発言が不自然に長くなったり、全候補を列挙したりしないようにする。
- 既に CO 結果を出す必要がある発言では、結果公開を優先し、ペア推理は最も効く 1 点だけにする。

推奨文面例:
- `疑い先を出す場合は、単体黒要素だけでなく、その人が人狼なら相方候補は誰か、2 人狼セットとして票筋・噛み筋が自然かも必要に応じて短く触れてください。`
- `全候補を長く列挙せず、今の結論に効く根拠 1〜2 点だけを出してください。`

注意:
- `task_daytime_speech()` は全役職が見る可能性があるため、人狼専用の確定相方情報や襲撃連携指示は入れない。
- `相方候補` は公開情報ベースの推理語として使ってよい。

## 6. 投票 task に 2 人狼セット比較を追加する

`task_vote()` を更新すること。

共通 task に追加する趣旨:
- 投票先は単体で怪しいかだけでなく、その人が狼なら相方候補が誰か、2 人狼セットとして票筋・発言・判定・噛み筋が自然かも考える。
- 投票理由は長く出さないが、最終判断では相方候補との整合を加味する。
- 候補トークン厳密一致、`intent=vote` / `intent=skip` の既存仕様は維持する。

wolf-only vote block に追加する趣旨:
- 既存の相方投票 discipline を維持しつつ、村から見た 2 人狼セットとしての自然さをさらに明示する。
- 相方を救う票、身内票、ライン切り、票逸らしは、翌日以降に自分と相方のペアとしてどう読まれるかを比較する。
- 相方情報は `role is Role.WEREWOLF` かつ `wolf_partner_tokens` がある場合だけ出す。

推奨文面例:
- 共通: `投票先は単体黒要素だけでなく、その人が人狼なら相方候補は誰か、2 人狼セットとして票筋・噛み筋が自然かも考えて選んでください。`
- wolf-only: `相方を救う票・切る票・票逸らしは、翌日以降に自分と相方の 2 人狼セットとして読まれても説明できるかを確認してください。`
```
